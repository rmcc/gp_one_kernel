Card Function Drivers

SDIO Functions

The SDIO stack at the top most level consists of function specific software modules, hereafter called SDIO function drivers. A single SDIO card can have one or more functions with each driver operating independently and in most cases not aware of the "other" functions. This feature of the stack allows card vendors to mix and match I/O functions and reuse drivers without modification. The embedded stack's flexible plug and play architecture makes this possible. SDIO function drivers utilize the SDIO Core's function driver interface to control their respective function.

SDIO Plug and Play

SDIO is a “plug and play” architecture with methods for an operating system to load appropriate control software based on information located in the card’s common register set or Card Information Structure (CIS). The CIS is a non-volatile memory region, accessible byte-wide, containing card or function specific meta-data. Each card has a common CIS along with one function specific CIS for each function on the card. The SDIO Core plug and play manager deals with information in the common CIS for manufacturer/card ID and power capabilities. For cards that support SDIO specification 1.10, the SDIO core will read manufacturer/card identifiers from the function CIS. A function driver may be interested in the function CIS for application specific parameters (i.e. MAC address, function capabilities, etc.). The SDIO core enumerates card functions and notifies function drivers when a compatible function is detected. more information on this mechanism can be found in the Developing Card Function Drivers section.

Control software can also be pre-installed into a non-volatile memory portion on the card called the CSA (Code Storage Area) on the card, however due to the different operating systems that a card may encounter, this feature is not supported.

SDIO defines standard device classes that expose a known register set and card behavior for control software (drivers). An operating system can support standard devices from a range of card manufacturers using a single class driver. The embedded stack provides class drivers for SD Memory, GPS and Bluetooth.
