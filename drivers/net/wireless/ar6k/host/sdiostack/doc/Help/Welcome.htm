Embedded SDIO Driver Kit Help

Welcome

Version 2.0:

This document contains help information for the Codetelligence Embedded SDIO Stack and Driver Kits (PDK or HDK).

Before using the software please read the end user license agreement.

Target System Requirements:

Linux:

x86 PC Platform running Fedora Core 3 (Kernel 2.6.9)
Tokyo Electron PCI-based "Ellen" Host Controller

PXA255 (XScale) Gumstix Platform (1-bit only) running Gumstix Distribution (Kernel 2.6.9RC1)

OMAP 1610 H2 Development Platform (Kernel 2.6.9 or Kernel 2.4.20)

PXA270 (XScale) SandGate IIp Development platform from Sophia Systems, (Kernel 2.6.9 w/ Sandgate IIp patches)

For other embedded operating systems, contact Codetelligence Sales (sales@codetelligence.com)

Development System Requirements:

Linux Development:

x86 :
Fedora Core 3 with GNU gcc 3.4.2.
Fedora Core 3 Linux Kernel Source.

PXA255:
Gumstix Build Environment (see www.gumstix.com) (kernel 2.6.9RC1)

OMAP:
2.6.9 kernel source for OMAP16xx (linux.omap.com) , requires gcc 3.3.2
2.4.20 kernel development requires MontaVista CEE 3.1 (OMAP1610H2)

PXA270:
Linux 2.6.9 kernel source w/ patches for SandGate IIp (contact Sophia Systems, www.sophia.com), requires gcc 3.4.3

IMPORTANT NOTE: For Linux 2.6 or higher kernels, the kernel must be configured with Plug and Play support in order for the bus driver to load. This requires a modification of the Kconfig file in \drivers\pnp. Refer to the FAQ for more information

It is highly recommended that you maximize the web browser or help application window when viewing the on-line users manual.
Where numbers are used a 0x prefix indicates a hexadecimal number. Binary numbers are post-fixed with the letter 'b' as in 10000001b.
Code samples and function names are displayed in a courier font.

The help manual can be navigated using the table of contents or by using the left and right arrows on each topic. The left arrow switches to the previous topic in the fixed chain of topics, while the right arrow moves to the next topic.
